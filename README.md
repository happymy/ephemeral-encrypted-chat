# 🔐 端到端加密匿名聊天室

一个基于 Cloudflare Workers 和 Durable Objects 的完全匿名、端到端加密实时聊天应用。  
消息在浏览器内使用 AES-256-GCM 加密，服务器无法解密。  
频道在所有人退出或长时间不活动后自动销毁，不留任何记录。

## ✨ 特性

- **端到端加密**：所有消息在本地加密/解密，密钥通过链接分享，服务器不可见
- **零存储**：不保存任何聊天记录、用户信息，消息仅保存在当前页面内存中
- **匿名即用**：无需注册、登录，分享链接即可加入加密频道
- **自动销毁**：频道内所有用户断开后 5 秒自动销毁；长时间无新消息也会自动销毁（默认 5 分钟，可配置）
- **手动销毁**：任意用户可一键立即销毁频道
- **自定义昵称**：支持设置显示名称，自动分配颜色，区分不同发言者
- **加入通知**：新用户加入时广播系统消息和弹出通知条
- **速率限制**：每连接每秒最多 3 条消息，超限会收到提示
- **安全加固**：已修复 CSS 注入、模板注入等常见前端和后端漏洞，配备安全 HTTP 头

## 🚀 快速部署

### 准备工作

需安装 Node.js (v18+) 和 Wrangler CLI，并登录 Cloudflare 账号。

### 配置与部署

1. 将项目文件（`wrangler.toml`、`src/index.js`、`src/chat-room.js`）放入同一目录。
2. 编辑 `wrangler.toml`，设置必要的环境变量：
   - `CHAT_PATH`：自定义聊天页面路径，默认为 `/chat`。
   - `INACTIVITY_TIMEOUT_SECONDS`：无新消息自动销毁超时秒数，默认 `300`（5 分钟），设为 `0` 可禁用。
   - `CLEAR_ON_DESTROY`：销毁频道后是否清空页面内容，默认 1（清空），设为 0 保留聊天记录。
   - `CLEAR_ON_DISCONNECT`：WebSocket 断开连接时是否立即清空页面历史消息，默认 1（清空），设为 0 保留聊天记录。
3. 执行 `wrangler deploy` 部署到 Cloudflare Workers。
4. 部署完成后，访问分配的 `*.workers.dev` 域名并加上聊天路径即可使用。

## 📖 使用方法

1. 打开网页，自动生成一个加密房间，地址栏会显示带密钥的链接。
2. 可设置昵称，随后进入聊天室。
3. 将完整链接通过安全渠道发送给他人，对方打开即可加入同一频道。
4. 所有消息在发送前加密，接收方自动解密，服务端无法查看内容。
5. 关闭页面或点击 💣 按钮可立即销毁频道；若无新消息且超时，系统会提前 30 秒倒计时提示并自动销毁。

> ⚠️ **重要提示**：链接中包含加密密钥，请通过安全渠道分享，避免公开传播。

## 🧱 架构

浏览器通过 WebSocket 连接到 Durable Object 房间实例，房间仅负责转发加密消息，不解析、不存储。

- **前端**：自包含 HTML 页面，完成密钥生成、加解密、UI 交互。
- **Worker**：处理 HTTP 请求，返回聊天页面或伪装页，并将 WebSocket 升级路由到对应的 Durable Object。
- **Durable Object**：管理房间连接，广播消息，执行销毁逻辑。

关键设计点：
- 房间 ID 和 256 位 AES 密钥编码在 URL 的 hash 中 (`#房间ID:base64密钥`)。
- 服务端完全不接触密钥，无法解密消息。
- 房间在无连接后 5 秒自动清除；无新消息超时（默认 5 分钟）后自动销毁，超时前有 30 秒警告倒计时。
- 支持手动销毁指令，立即关闭所有连接并清空数据。

## 🔒 安全审计

本项目经过多轮安全审计，当前状态：

- ✅ **CSS 注入**：用户自定义颜色严格校验为 `#XXXXXX` 格式。
- ✅ **模板注入**：前端脚本中的路径使用安全转义。
- ✅ **XSS 防护**：所有用户输入输出均经过 HTML 实体编码。
- ✅ **速率限制**：每连接每秒最多 3 条消息，超限时发送方会收到提示。
- ✅ **安全响应头**：已设置 `X-Content-Type-Options: nosniff`、`X-Frame-Options: DENY` 及内容安全策略。
- ✅ **销毁逻辑防重入**：频道销毁过程中防止重复关闭连接和残留定时器。
- ⚠️ **密钥传输**：加密密钥通过 URL hash 分享，这是端到端加密的必要权衡，请务必通过安全渠道分享链接。
- ⚠️ **重放攻击**：消息不含时间戳，存在一定重放可能（需攻击者突破 TLS）。
- ⚠️ **CSP 内联脚本**：为保持自包含单文件架构，允许内联脚本，但所有外部输入均已严格消毒。
